XCAT Node Discovery

Node discovery is not working yet, we manually looked up the MAC addresses on the NICs on the compute nodes.

Node Discovery

Detect the BMCs and add the node definitions into xCAT.
Note: The bmcdiscover command will use the username/password from the passwd table corresponding to key=ipmi.

[root@se_m1 ~]# bmcdiscover --range 172.29.101.1-32 -z -w > ./bmc.stanza

Optional: Order the machines in the bmc.stranza file. For example, using vim:
[root@se_m1 ~]# vim ./bmc.stanza
/101.2 # search for 101.2
10 dd # cut 10 lines ()
:10 # go to line 10
p # paste the 10 lines
... # repeat till all are sorted.

Edit the bmc.stanza file, change the node’s hostname and add its IP.
[root@se_m1 ~]# cat ./bmc.stanza
n001:
objtype=node
groups=all
bmc=172.29.101.1
cons=ipmi
mgt=ipmi
mtm=5465AC1
serial=J11YZKX
ip=10.10.2.1
...

Define the compute nodes into xCAT:
[root@se_m1 ~]# cat bmc.stanza | mkdef -z

We can also update the node definition, using chdef command. For example, to add the IP of n001 using the chdef command.
[root@se_m1 ~]# chdef n001 ip=10.10.2.1

In order to do BMC configuration during the discovery process, set runcmd=bmcsetup.
[root@se_m1 ~]# for n in {001..032}; do chdef n${n} chain="runcmd=bmcsetup"; done

Then, add node info into /etc/hosts and DNS:
[root@se_m1 ~]# for n in {001..032}; do makehosts n${n} ; done
[root@se_m1 ~]# makedns -n

Start discovery process

To start discovery process, just need to power on the host remotely with the following command, and the discovery process will start automatically after the host is powered on:
[root@se_m1 ~]# for n in {001..032}; do rpower n${n} on; done

[Optional] Monitor the discovery process, you can use:
[root@se_m1 ~]# chdef n001 cons=ipmi
[root@se_m1 ~]# makeconservercf
[root@se_m1 ~]# rcons n001
