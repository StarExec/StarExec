xCAT Configuration

I. Configure the Management Node

By setting properties on the Management Node before installing the xCAT software will allow xCAT to automatically configure key attributes in the xCAT site table during the install.

Ensure a hostname is configured on the management node by issuing the hostname command. [It’s recommended to use a fully qualified domain name (FQDN) when setting the hostname]

1. To set the hostname of se_m1.ccs.miami.edu:
[root@se_m1 ~]# hostnamectl set-hostname se_m1.ccs.miami.edu

2. Add the hostname to the /etc/sysconfig/network in order to persist the hostname on reboot.
[root@se_m1 ~]# cat >> /etc/sysconfig/network << EOF
HOSTNAME=se_m1.ccs.miami.edu
EOF

3. Automatic Install Using go-xcat
go-xcat is a tool that can be used to fully install or update xCAT. go-xcat will automatically download the correct package manager repository file from xcat.org and use the public repository to install xCAT.

[root@se_m1 ~]# wget https://raw.githubusercontent.com/xcat2/xcat-core/master/xCAT-server/share/xcat/tools/go-xcat -O - >/tmp/go-xcat
# chmod +x /tmp/go-xcat
# /tmp/go-xcat install

4. Verify the Install
[root@se_m1 ~]# source /etc/profile.d/xcat.sh
# tabdump site

5. Configure the xCAT site table. Leave default values, updated fields below.

[root@se_m1 ~]# tabedit site

"nameservers","10.10.2.254",,
"dhcpinterfaces","eno1"

"tftpdir","/tftpboot",,
"xcatdport","3001",,

"timezone","America/New_York",,

...

7. Setup the xCAT noderes table
The noderes table defines the resources of the nodes. This includes all of the servers it uses to boot to a usable state and all the types of boot ups it will do.
Note: Primary interface (primarynic) for ipmi machines is the one where the BMC is set to.

[root@se_m1 ~]# chdef -t group -o compute netboot=pxe tftpserver=10.10.2.254 nfsserver=10.10.2.254 installnic=eno1 primarynic=eno1

8. Setup the xCAT passwd table
[root@se_m1 ~]# chtab key=system passwd.username=root passwd.password=sexec-clust3r1
[root@se_m1 ~]# chtab key=ipmi passwd.username=USERID passwd.password=PASSW0RD
[root@se_m1 ~]# tabedit passwd
#key,username,password,cryptmethod,authdomain,comments,disable
"system","root","sexec-clust3r1",,,,
"ipmi","USERID","PASSW0RD",,,

9. Setup the xCAT chain table
[root@se_m1 ~]# tabedit chain
#node,currstate,currchain,chain,ondiscover,comments,disable
"n00",,,,,,
"n01",,,,,,
"n02",,,,,,
...
"n32",,,,,,

10. Setup the xCAT nodetype table
[root@se_m1 ~]# tabedit nodetype
#node,os,arch,profile,provmethod,supportedarchs,nodetype,comments,disable
"n00","centos5.4","x86_64","compute",,,,,
"n01","centos5.4","x86_64","compute",,,,,
...
"n32","centos5.4","x86_64","compute",,,,,

11. Setup the xCAT hosts table
[root@se_m1 ~]# tabedit hosts
xcat,(xcat_int_ip)
"n00",(n00_ip)
"n01",(n01_ip)
...
"n32",(n02_ip)

12. Setup the xCAT mac table
We manually looked up these MAC addresses
[root@se_m1 ~]# tabedit mac
"n00","eno1",(mac of en01)
"n01","eno1",(mac of eno1)
...
"n32","eno1",(mac of eno1)

13. Setup the xCAT nodelist table
[root@se_m1 ~]# tabedit nodelist
"n00","compute,all",,,
"n01","compute,all",,,
...
"n32","compute,all",,,

14. Setup the xCAT nodehm table
[root@se_m1 ~]# tabedit nodehm
"n00","ipmi","ipmi",,,,,,,,,,
"n01","ipmi","ipmi",,,,,,,,,,
...
"n32","ipmi","ipmi",,,,,,,,,,

15. Create the hosts file
[root@se_m1 ~]# makehosts all

There should now be entries in the /etc/hosts that reflect all the nodes

16. Create the DHCP files
[root@se_m1 ~]# makedhcp -n
makedhcp all
service dhcpd restart
chkconfig --level 345 dhcpd on

17. Edit /etc/resolv.conf
[root@se_m1 ~]# vi /etc/resolv.conf
search cluster.edu
nameserver 10.10.2.254

18. Build the DNS server
[root@se_m1 ~]# makedns
makedns all
service named restart
chkconfig --level 345 named on

19. Routing to the Internet through the Head Node
[root@se_m1 ~]# echo "iptables -t nat -A POSTROUTING -o eno1 -j MASQUERADE" >> /etc/rc.local
echo "echo 1 > /proc/sys/net/ipv4/ip_forward" >> /etc/rc.local

Remember to configure the gateway for each of the compute node to eno1 (private nic) of the Head Node.
Go to client private nic /etc/sysconfig/network-scripts/ifcfg-eno1 and add “GATEWAY=10.10.2.254”

The xCAT server should now be completely configured.

20. Setup Images
[root@se_m1 ~]# wget http://ftp.usf.edu/pub/centos/7.4.1708/isos/x86_64/CentOS-7-x86_64-DVD-1708.iso
# copycds /root/CentOS-7-x86_64-DVD-1708.iso

21. Install the nodes
[root@se_m1 ~]# for n in {001..032}; do chdef -t node -o n${n} netboot=pxe; done
[root@se_m1 ~]# for n in {019..023}; do rinstall n${n} osimage=centos7.4-x86_64-install-compute; done
